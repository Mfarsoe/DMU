Relative enheder
Tilpasser sig omgivelserne

Relative enheder ændrer sig afhængigt af noget andet – skærmstørrelse, forældreelement eller brugerens indstillinger. De er rygraden i responsivt design.

%
Procent
Relativ til forældreelement
bredde højde padding
▾

Hvad er det?

Procent er altid relativ til forælderelementet. Hvis din container er 800px bred, bliver width: 50% til 400px. Padding og margin i % beregnes altid ud fra bredden af forælderen — også for top/bottom padding.

.container {
  width: 80%; /* 80% af forælder */
}

.child {
  width: 50%; /* 50% af .container */
  padding: 5%;
}
✓ Brug til
  • Fleksible layouts og kolonner
  • Billeder der skalerer med forælder
  • Responsive containere
✗ Undgå til
  • Skriftstørrelser (brug rem i stedet)
  • Præcise mål der aldrig skal ændre sig
vw / vh
Viewport Width / Height
Relativ til browservinduet
hero fullscreen skriftstørrelse
▾

Hvad er det?

1vw = 1% af browservinduets bredde. 1vh = 1% af browservinduets højde. 100vw er altid hele skærmbredden. Perfekt til elementer der fylder hele skærmen, uanset enhed.

.hero {
  width: 100vw;
  height: 100vh; /* Fylder skærmen */
}

h1 {
  font-size: 5vw; /* Skalerer med vinduet */
}

clamp()
Clamp — flydende størrelse
Min, foretrukken, max — alt i én linje
typografi spacing layout
▾

Hvad er det?

clamp(min, foretrukken, max) lader dig sige: "Vær mindst X, helst Y (flydende), men aldrig mere end Z." Ekstremt kraftfuldt til responsiv typografi — ingen media queries nødvendige!

h1 {
  font-size: clamp(24px, 5vw, 72px);
  /* Min: 24px | Flydende: 5vw | Max: 72px */
}

.container {
  padding: clamp(16px, 4vw, 60px);
}
✓ Brug til
  • Overskrifter der skalerer smukt
  • Responsiv spacing uden media queries
  • Moderne, flydende layouts
✗ Undgå til
  • Meget simple sider (overkill)
  • Når du har brug for hårde breakpoints
Absolutte enheder
Fast og præcis

Absolutte enheder er altid det samme, uanset skærmstørrelse eller indstillinger. Brug dem kun når du har brug for præcis kontrol — typisk til ikoner, borders og print.

px
Pixel
Den mest brugte absolutte enhed
border border-radius ikoner
▾

Hvad er det?

En pixel er en fast enhed — den ændrer sig ikke. Den er faktisk ikke direkte en skærmspixel på moderne enheder (high-DPI skærme), men opfører sig konsistent på tværs af browsere. God til præcise detaljer, men undgå den til skriftstørrelser og layout — det bryder tilgængelighed.

.card {
  border: 1px solid #333;
  border-radius: 8px;
  box-shadow: 0 2px 12px rgba(0,0,0,0.2);
}

img.icon {
  width: 24px;
  height: 24px;
}
✓ Brug til
  • Borders og outlines (1px, 2px)
  • Border-radius på knapper/kort
  • Ikoner og faste billeder
  • Box-shadow værdier
✗ Undgå til
  • Skriftstørrelser — bryder brugerens zoomindstillinger
  • Overordnet layout (brug %, rem, vw)
Skrift-relative enheder
Bygget til typografi

Disse enheder er relative til skriftstørrelse — enten rodelementets (rem) eller forælderelementets (em). De er fundamentale for tilgængeligt og vedligeholdbart design.

rem
Root Em
Relativ til <html> — den vigtigste enhed
font-size spacing layout
▾

Hvad er det?

1rem = skriftstørrelsen på <html>-elementet. Browseren sætter denne til 16px som standard. Ændrer brugeren sin browserindstilling, skalerer alt i rem med. Det er den vigtigste enhed for tilgængelighed.

/* html = 16px standard */

h1 { font-size: 2rem; } /* = 32px */
p { font-size: 1rem; } /* = 16px */
small { font-size: 0.75rem; } /* = 12px */

.section {
  padding: 2rem 1.5rem;
}
✓ Brug til
  • Alle skriftstørrelser
  • Padding og margin i layouts
  • Gap i grids og flex
  • Alt der skal respektere brugerens indstillinger
✗ Undgå til
  • Borders — px er finere her
  • Meget præcise grafiske detaljer
em
Em
Relativ til forælderens skriftstørrelse
padding i komponenter nested tekst
▾

Hvad er det?

1em = skriftstørrelsen på det nuværende element (arvet fra forælder). Det kan føre til kompounderende effekter i nestede elementer (em × em × em...). Nyttigt til padding på knapper og komponenter der skal skalere med deres egen skriftstørrelse.

.button {
  font-size: 1rem;
  padding: 0.6em 1.2em; /* relativt til knappens font */
}

.button.large {
  font-size: 1.25rem;
  /* padding skalerer automatisk! */
}
✓ Brug til
  • Padding på knapper og tags
  • Komponenter der skal skalere internt
  • Letter-spacing og line-height
✗ Undgå til
  • Globale layouts (brug rem)
  • Indlejrede strukturer — kompounderer

🏆 Tommelfingerregel

Brug rem til skriftstørrelser og spacing, % til layout og bredder, px til borders og detaljer, og vw/vh til fuld-skærmselementer. Undgå px til skriftstørrelser — det bryder tilgængelighed.
