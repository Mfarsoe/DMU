Box Model

Hvert HTML-element er en boks. Box model beskriver, hvordan dens plads beregnes: indhold, padding, border og margin. At forstå dette er fundamentet for alt CSS-layout.

margin
border
padding
content
De fire lag
Hvordan elementers størrelse og afstand beregnes
marginpaddingborder
▾

De fire lag

Content — det faktiske indhold (tekst, billeder)
Padding — afstand mellem indhold og border (transparen, men farvet med baggrundsfarven)
Border — kanten rundt om elementet
Margin — afstand til andre elementer (altid transparent)

box-sizing: border-box er afgørende — den sørger for at padding og border er inkluderet i elementets bredde, ikke adderet ovenpå.

Visualisering
margin
border
padding
content
/* ALTID sæt dette globalt — gem det udenad */
*, *::before, *::after {
  box-sizing: border-box;
}

.boks {
  width: 300px; /* total bredde inkl. padding + border */
  padding: 20px; /* luft indeni på alle sider */
  border: 2px solid #333; /* synlig kant */
  margin: 16px auto; /* udenfor + centrering med auto */
}

/* Individuelle sider: top right bottom left */
.asymmetrisk {
  margin: 8px 16px 24px 0;
  padding-top: 32px;
  border-bottom: 1px solid #444;
}
✓ Brug
  • Sæt altid box-sizing: border-box globalt
  • Brug margin: 0 auto til at centrere blok-elementer
  • Padding til intern luft, margin til afstand mellem elementer
✗ Undgå
  • Negativ margin med forsigtighed (kan overlappe)
  • Margin på inline-elementer (virker ikke altid)
  • Ikke brug padding til afstand mellem elementer — brug gap i flex/grid

Selektorer

Selektorer bestemmer hvilke HTML-elementer dine CSS-regler gælder for. Der er mange typer — fra simple element-selektorer til kraftfulde kombinations-selektorer.

element
.klasse
#id
Basis-selektorer
De tre fundamentale typer
element.class#id
▾

De tre grundtyper

Element-selektor — vælger alle elementer af en type: p, h1, div

Klasse-selektor — vælger elementer med en bestemt klasse: .kortliste

ID-selektor — vælger ét unikt element: #menu. Brug sparsomt — høj specificitet og genbruges ikke.

Du kan kombinere: p.intro = p-elementer med klassen "intro".

/* Element */
p { color: #333; }

/* Klasse — kan bruges mange gange */
.kort { border-radius: 8px; }

/* ID — kun én gang per side */
#header { position: sticky; }

/* Universel — alle elementer */
* { box-sizing: border-box; }

/* Kombination */
p.ingress { font-size: 1.2rem; }

Kombinations-selektorer
Vælg elementer baseret på deres relation til andre

▾

Relationsselektorer

A B — efterkommer: alle B inde i A (uanset dybde)

A > B — direkte barn: kun B der er direkte under A

A + B — næste søskende: B der kommer lige efter A

A ~ B — alle søskende: alle B efter A på samme niveau

A, B — komma = "og": stil A og B ens

/* Alle li inde i .nav (alle niveauer) */
.nav li { list-style: none; }

/* Kun direkte h2-børn af .sektion */
.sektion > h2 { font-size: 1.5rem; }

/* p direkte efter h2 */
h2 + p { margin-top: 0; }

/* Alle .aktiv efter .markeret */
.markeret ~ .aktiv { opacity: .5; }

/* Stil h1 og h2 ens */
h1, h2 { font-family: serif; }

Attribut-selektorer

Vælg elementer baseret på tilstedeværelse eller indhold af HTML-attributter.

  • [href] — har en href-attribut
  • [type="email"] — nøjagtigt match
  • [class*="btn"] — indeholder "btn"

}

/* Data-attributter */
[data-status="aktiv"] {
  background: green;
}

CSS Custom Properties

CSS-variabler (custom properties) lader dig gemme værdier der kan genbruges overalt. Essentielle til temaer, dark mode og vedligeholdelig CSS.

--navn
var(--navn)
:root { }
Custom Properties
Definer én gang, brug overalt
:rootvar()fallback
▾

Hvad er det?

Custom properties starter altid med --. De defineres typisk på :root (globalt scope) og bruges med var(--navn).

De er dynamiske — JavaScript kan ændre dem, og de nedarves i DOM-træet. Du kan have fallback-værdier: var(--farve, #333).

Perfekt til farvetemaer og dark mode!

/* Globale variabler på :root */
:root {
  --farve-primær: #3b82f6;
  --farve-tekst: #1a1a1a;
  --radius: 8px;
  --skygge: 0 2px 12px rgba(0,0,0,.15);
}

.knap {
  background: var(--farve-primær);
  border-radius: var(--radius);
  box-shadow: var(--skygge);
}

/* Dark mode — overskriver blot :root */
@media (prefers-color-scheme: dark) {
  :root { --farve-tekst: #f0f0f0; }
}
✓ Brug til
  • Farvepaletter og temaer
  • Gentagne spacing-værdier
  • Dark/light mode med @media
  • Animations-parametre styret af JS
✗ Undgå
  • Brug ikke til ét-gangs værdier
  • Navngivning bør være semantisk (--farve-primær ikke --blå)

Positioning

Position-egenskaben bestemmer, hvordan et element placeres i dokumentet. Det er fundamentalt at forstå de fem typer og hvornår man bruger dem.

static
relative
absolute
fixed
sticky
De fem position-værdier
Hvordan elementer placeres i forhold til deres kontekst
top/leftz-index
▾

Positioneringstyper

static — standard. Elementet følger normal flow. top/left virker ikke.

relative — forskyder elementet fra dets normale position. Skaber en positioning context for children.

absolute — trækkes ud af flow. Placeres relativt til nærmeste positioned forfader.

fixed — fastgjort til browservinduet. Bevæger sig ikke ved scroll.

sticky — opfører sig som relative indtil scroll-grænsen nås, derefter som fixed.

/* Relative — skubbes 10px ned fra normal pos */
.badge {
  position: relative;
  top: -2px; /* fin justering */
}

/* Absolute inde i relative forælder */
.kort { position: relative; }
.kort .hjørne-badge {
  position: absolute;
  top: 8px; right: 8px;
}

/* Fixed header */
.topbar {
  position: fixed;
  top: 0; left: 0; right: 0;
}

/* Sticky tabel-header */
th {
  position: sticky;
  top: 0;
}

Float & Clearfix

float: left/right trækker elementet ud af normal flow. Omgivende tekst og inline-elementer flyder rundt om det.

Clearfix-problemet: En container der kun indeholder floated elementer, kollapser til 0 højde. Løsning: brug overflow: auto/hidden på containeren, eller ::after { clear: both }.

I dag bruges floats primært til billeder i artikler.

img.artikel-billede {
  float: left;
  margin: 0 1.5rem 1rem 0;
  max-width: 240px;
  border-radius: 6px;
}

/* Clearfix — klassisk løsning */
.clearfix::after {
  content: "";
  display: block;
  clear: both;
}

/* Moderne løsning */
.container { overflow: hidden; }

CSS Grid

Grid er CSS's kraftigste layout-system — det styrer både rækker og kolonner samtidigt. Perfekt til sideopbygning, gallerier og komplekse 2D layouts.

display:grid
grid-template
fr / minmax()
grid-area
Grid-systemet
Rækker, kolonner, fr-enheden og placement
▾

Grid-begreber

fr — "fraction unit". 1fr = én andel af ledig plads. 1fr 2fr = 1/3 og 2/3.

repeat() — gentag kolonner: repeat(3, 1fr)

minmax() — angiv min og max for en kolonne

auto-fill/auto-fit — auto-antal kolonner, responsivt uden media queries

grid-area — navngiv og placer elementer præcist

/* Simpelt 3-kolonne grid */
.grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 1.5rem; }

/* Responsivt uden media queries */
.auto {
  grid-template-columns:
    repeat(auto-fill, minmax(200px, 1fr));
}

/* Navngivet layout */
.side {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main "
    "footer footer ";
  grid-template-columns: 260px 1fr;
}
header { grid-area: header; }

Overflow & Display

Display bestemmer, hvordan et element opfører sig i layoutet. Overflow styrer, hvad der sker, når indholdet er større end containeren.

block
inline
inline-block
none
Display-værdier
Fundamentalt for elementers opførsel i flow
▾

De vigtigste værdier

block — fylder fuld bredde, starter på ny linje (div, p, h1)

inline — følger tekstflow, margin/width virker ikke horisontalt (span, a)

inline-block — som inline, men accepterer width/height og vertical margin

none — fjerner elementet helt (optager ingen plads)

flex / grid — se egne sektioner

contents — elementet selv "forsvinder", men children forbliver

/* Skjul element — fjerner det fra layout */
.skjult { display: none; }

/* Gør link til blok (fylder fuld bredde) */
a.menu-link { display: block; }

/* inline-block — bruges til knapper */
.badge {
  display: inline-block;
  padding: .25rem .75rem;
}

/* visibility: hidden — gemmer men beholder plads */
.usynlig { visibility: hidden; }
/* opacity: 0 — transparent, men stadig i layout */
.gennemsigtig { opacity: 0; }
/* overflow */
.boks { overflow: hidden; } /* klip indhold der løber ud */
.scroll { overflow: auto; } /* scroll kun ved behov */
.scroll-x { overflow-x: auto; overflow-y: hidden; }

/* Afkort tekst med ellipsis */
.ét-linje {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

Z-index & Stacking Context

Z-index bestemmer hvilke elementer der tegnes ovenpå andre. Men der er en hage: z-index virker kun på positioned elementer, og stacking contexts kan forvirre.

z-index
stacking
context
Z-index
Hvem vises oven på hvem
▾

Regler

z-index virker kun på elementer med position ≠ static (eller flex/grid items).

Højere z-index = visually oven på. Standard = auto.

Stacking context: Visse egenskaber skaber en ny stacking context (opacity < 1, transform, filter, isolation: isolate). Elementer inde i én context kan ikke "slippe ud" af den.

Brug isolation: isolate bevidst for at afgrænse z-index-scope.

/* Typisk z-index skala */
.dropdown { z-index: 100; position: absolute; }
.modal { z-index: 500; position: fixed; }
.overlay { z-index: 490; position: fixed; }
.tooltip { z-index: 1000; position: absolute; }

/* isolation: isolate — begræns z-index scope */
.kort-container { isolation: isolate; }

/* CSS-variabler til z-index skala */
:root {
  --z-dropdown: 100;
  --z-modal: 500;
  --z-toast: 1000;
}

Transform & Filter

Transform lader dig flytte, rotere, skalere og skæve elementer visuelt — uden at påvirke layout. Filter giver visuelle effekter som blur og brightness.

translate
rotate
scale
skew
Transform-funktioner
Flyt, roter og skaler uden at bryde layout
▾

Transform-funktioner

translateX/Y(n) — flyt vandret/lodret

rotate(ndeg) — roter

scale(n) — skaler (1 = original)

skewX/Y(ndeg) — skæv

Kombinér: transform: rotate(45deg) scale(1.2)

transform-origin — ændr rotation-omdrejningspunkt

Transforms udføres af GPU — brug dem til animationer frem for top/left!

/* Hover-effekt — løft knap */
.knap:hover {
  transform: translateY(-2px);
}

/* Roter ikon */
.pil.åben {
  transform: rotate(180deg);
}

/* Centrering med transform */
.modal {
  position: fixed;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%);
}

/* filter — visuelle effekter */
.uklar { filter: blur(4px); }
.gråtone { filter: grayscale(100%); }
.lys { filter: brightness(1.3); }

Matematiske Funktioner

CSS har kraftfulde matematiske funktioner der lader dig beregne, begrænse og kombinere værdier dynamisk — direkte i CSS.

clamp()
min() / max()

CSS Math
Dynamiske værdier med logik direkte i CSS
▾

Funktioner

clamp(min, val, max) — begræns til område. Perfekt til fluid typografi.

min(a, b) — vælg mindste. min(100%, 800px) = responsiv container.

max(a, b) — vælg største. max(1rem, 2vw) = tilgængelig skrift.

calc(udtryk) — beregn. Kan blande enheder: calc(100% - 2rem).

round(), mod(), rem() — nyere funktioner til præcis matematik.

/* Fluid typografi — ingen media queries! */
h1 { font-size: clamp(1.5rem, 5vw, 4rem); }

/* Responsiv container */
.wrap { width: min(100%, 960px); margin: 0 auto; }

/* Sikr minimum skriftstørrelse */
p { font-size: max(1rem, 1.5vw); }

/* Beregn bredde */
.sidebar-main {
  width: calc(100% - 260px - 2rem);
}

/* Fluid spacing-skala */
:root {
  --space-m: clamp(1rem, 3vw, 2rem);
  --space-l: clamp(2rem, 6vw, 4rem);
}

CSS Reset & Normalize

Browsere har deres egne standard-styles for HTML-elementer. Et reset eller normalize fjerner eller udjævner disse forskelle, så du starter fra et forudsigeligt udgangspunkt.

* { margin:0 }
normalize
modern reset
Reset vs. Normalize
Fjern browser-defaults for konsistent udgangspunkt
cross-browserdefaults
▾

To tilgange

CSS Reset — fjerner ALT. Margin, padding, border-styles, font-størrelser m.m. Du bygger alt op selv. Eric Meyer's reset er klassikeren.

Normalize.css — udjævner forskelle, men beholder nyttige browser-defaults. Fx. bevarer den h1's store skrift, men gør den konsistent på tværs af browsere.

Modern reset — moderne tilgang der tager det bedste fra begge. Inkluderer vigtige accessibility-fixes og typiske best-practices.

/* ── Modern CSS Reset (anbefalet udgangspunkt) ── */
*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}
html {
  -webkit-text-size-adjust: 100%; /* iOS rotation fix */
  scroll-behavior: smooth;
}
body {
  min-height: 100vh;
  line-height: 1.6;
  -webkit-font-smoothing: antialiased;
}
img, picture, video, canvas, svg {
  display: block;
  max-width: 100%; /* aldrig bredere end container */
}
input, button, textarea, select {
  font: inherit; /* form-elementer arver ikke font som standard */
}
p, h1, h2, h3, h4, h5, h6 {
  overflow-wrap: break-word; /* undgå horisontalt overflow */
}
ul[role="list"], ol[role="list"] {
  list-style: none; /* fjern bullets fra navigations-lister */
}
✓ Altid inkluder
  • box-sizing: border-box på alle elementer
  • max-width: 100% på billeder
  • font: inherit på form-elementer
  • line-height på body
✗ Pas på
  • Et fuldt reset nulstiller OGSÅ nyttige ting som :focus-outline
  • Husk at genindsætte focus-styles for tilgængelighed

CSS understøtter mange farveformater. At kende dem alle giver dig fuld kontrol over farver, gennemsigtighed og dynamiske farveberegninger.

Farveformater

#hex — #rrggbb eller kort form #rgb. Ingen alpha-kanal i standard hex, men #rrggbbaa understøttes i moderne browsere.

rgb() / rgba() — rød, grøn, blå (0–255). rgba(0,0,0,0.5) = halvgennemsigtig sort.

hsl() / hsla() — Hue (0–360°), Saturation (%), Lightness (%). Meget intuitivt til at lave farve-varianter.

oklch() — moderne, perceptuelt ensartet farverum. Bedre til at beregne kontrast-sikre farver og farveinterpolation i gradienter.

currentColor — arver elementets aktuelle color-egenskab. Nyttigt til borders og SVG-fills.

transparent — nøgleord for fuldt gennemsigtig.

/* Hex */
.el { color: #e83e8c; }
.el { color: #f00; } /* kort form = #ff0000 */

/* RGB / RGBA */
.el { background: rgb(59 130 246); } /* moderne syntax */
.el { background: rgba(0, 0, 0, 0.4); } /* 40% sort */

/* HSL — nemmest at forstå visuelt */
.el { color: hsl(210, 80%, 55%); }
.el-lys { color: hsl(210, 80%, 75%); } /* lysere variant */
.el-mrk { color: hsl(210, 80%, 35%); } /* mørkere variant */

/* OKLCH — moderne anbefaling */
.el { color: oklch(65% 0.2 240); } /* L% C H */

/* currentColor — arver color-egenskaben */
.ikon {
  color: royalblue;
  border: 2px solid currentColor; /* samme blå */
  fill: currentColor; /* SVG-fill følger med */
}

opacity vs. rgba — hvad er forskellen?

opacity: 0.5 gør elementet og ALT dets indhold gennemsigtigt — tekst, borders, children. Det kan ikke nedarves i omvendt retning.
rgba(0,0,0,0.5) eller hsl(0,0%,0%,50%) gør kun den specifikke farve gennemsigtig — f.eks. baggrund, mens teksten forbliver fuldt synlig.

Borders & Outline

Border og outline ligner hinanden, men opfører sig meget forskelligt. Border tager plads i box model. Outline gør ikke — og bruges primært til fokus-styling.

border
border-radius
border-image
outline
Border-egenskaber
Kanter, afrunding og fokus-ringe
border-radiusoutlineborder-image
▾

Border vs. Outline

border — en del af box model. Tager plads. Kan styles individuelt per side.

outline — tegnes uden for elementet. Tager ingen plads, forskydes ikke layout. Bruges til focus-rings — fjern aldrig outline: none uden alternativ!

border-radius — afrunder hjørner. 50% = cirkel. Kan sættes individuelt per hjørne.

border-image — brug et billede eller gradient som border.

outline-offset — afstand mellem elementet og outline-linjen.

/* Border shorthand: bredde stil farve */
.kort {
  border: 1px solid #ddd;
  border-radius: 12px;
}

/* Individuelle sider */
.input {
  border: none;
  border-bottom: 2px solid #ccc;
}
.input:focus { border-bottom-color: #3b82f6; }

/* Afrunding per hjørne: top-left top-right bottom-right bottom-left */
.pille { border-radius: 999px; } /* perfekt pille */
.cirkel { border-radius: 50%; } /* cirkel */
.asym { border-radius: 4px 16px 16px 4px; }

/* Outline til focus — tilgængelighed */
:focus-visible {
  outline: 2px solid #3b82f6;
  outline-offset: 3px;
}

/* Gradient border med border-image */
.fancy-border {
  border: 2px solid transparent;
  border-image: linear-gradient(90deg, #e83e8c, #3b82f6) 1;
}

Box Shadow & Text Shadow

Shadows tilføjer dybde og elevation til elementer. CSS understøtter både box-skygger på containere og text-skygger på tekst — med fuld kontrol over retning, spredning og farve.

box-shadow syntax

box-shadow: x y blur spread farve

  • x — vandret offset (positiv = højre)
  • y — lodret offset (positiv = nedad)
  • blur — sløring (0 = skarp)
  • spread — gør skyggen større/mindre
  • inset — skygge indeni elementet

Du kan stapele flere skygger med komma. filter: drop-shadow() følger elementets faktiske form (god til PNG med transparency).

/* Simpel skygge */
.kort { box-shadow: 0 2px 8px rgba(0,0,0,.12); }

/* Elevation-system (Material Design-stil) */
.lav { box-shadow: 0 1px 3px rgba(0,0,0,.12); }
.mid { box-shadow: 0 4px 16px rgba(0,0,0,.15); }
.høj { box-shadow: 0 12px 40px rgba(0,0,0,.2); }

/* Inset — fordybning / indre skygge */
.input { box-shadow: inset 0 2px 4px rgba(0,0,0,.1); }

/* Glød-effekt */
.glød { box-shadow: 0 0 20px rgba(59,130,246,.5); }

/* Stablede skygger — mere realistisk */
.naturlig {
  box-shadow:
    0 1px 1px rgba(0,0,0,.06),
    0 2px 4px rgba(0,0,0,.06),
    0 4px 8px rgba(0,0,0,.06);
}

/* text-shadow: x y blur farve */
h1 { text-shadow: 2px 2px 4px rgba(0,0,0,.3); }
.neon { text-shadow: 0 0 10px #e83e8c, 0 0 30px #e83e8c; }

Width, Height & Sizing

CSS har mange måder at kontrollere elementers størrelse — faste mål, fleksible, og min/max-begrænsninger. At forstå hvornår man bruger hvad er afgørende for godt layout.

width / height

Størrelses-egenskaber
Faste, fleksible og begrænsede størrelser
min-widthaspect-ratiofit-content
▾

Størrelsesegenskaber

width / height — fast størrelse. Undgå fast height på tekst-containere.

min-width / min-height — mindst X, må gerne vokse.

max-width / max-height — højst X, må gerne krympe.

aspect-ratio — bevar proportioner. Perfekt til billeder og video-wrappers.

width: fit-content — vær så bred som indholdet kræver.

width: min-content — krympet til mindste mulige bredde.

width: max-content — bred nok til indholdet uden linjeskift.

/* Responsiv container — klassisk pattern */
.container {
  width: min(100%, 1200px);
  margin-inline: auto; /* centrering */
  padding-inline: 1.5rem;
}

/* Aldrig kortere end 200px, aldrig bredere end 400px */
.kort {
  min-width: 200px;
  max-width: 400px;
}

/* aspect-ratio — bevar 16:9 */
.video-wrapper {
  width: 100%;
  aspect-ratio: 16 / 9;
}

/* fit-content — kun så bred som nødvendigt */
.tag { width: fit-content; }

/* Logiske egenskaber (block = lodret, inline = vandret) */
.el {
  block-size: 200px; /* = height */
  inline-size: 100%; /* = width */
  margin-block: 1rem; /* = margin-top + bottom */
  padding-inline: 2rem; /* = padding-left + right */
}

Spacing-strategier

Afstand er halvdelen af et godt design. CSS har mange måder at håndtere spacing — og der er klare regler for hvornår du bruger margin, padding, gap eller inset.

margin
padding
gap
inset
Spacing-egenskaber
Hvornår bruges margin, padding og gap?
margin: autogapmargin-collapse
▾

Hvornår hvad?

padding — luft indeni et element. Baggrund og border strækker sig ind i padding. Brug til intern luft i kort, knapper, sektioner.

margin — afstand mellem elementer. Transparent altid. margin: auto centrerer blok-elementer vandret. Pas på margin collapse — lodrette margins mellem blok-elementer smelter sammen til den største.

gap — afstand mellem flex- eller grid-items. Bedre end margin til dette formål, da det ikke tilføjer afstand på yderkanter.

inset — shorthand for top/right/bottom/left på positioned elementer.

/* Padding — intern luft */
.kort { padding: 1.5rem; }
.knap { padding: .6em 1.2em; }

/* Margin — afstand til andre elementer */
.afsnit { margin-bottom: 1rem; }
.centrer { margin-inline: auto; } /* centrering */

/* Gap — i flex/grid */
.flex-row { display: flex; gap: 1rem; }
.grid { display: grid; gap: 1rem 2rem; } /* row col */

/* inset — shorthand for positioned elementer */
.overlay {
  position: absolute;
  inset: 0; /* = top:0 right:0 bottom:0 left:0 */
}

/* Spacing-skala med CSS-variabler */
:root {
  --s-xs: .25rem; --s-sm: .5rem;
  --s-md: 1rem; --s-lg: 2rem;
  --s-xl: 4rem; --s-2xl: 8rem;
}

⚠️ Margin Collapse

Lodrette margins på blok-elementer smelter (collapper) — de adderes ikke, den største vinder. F.eks. en h2 med margin-bottom: 24px efterfulgt af et p med margin-top: 16px giver 24px afstand, ikke 40px. Collapse sker ikke ved: flex/grid-containers, floated elementer, eller overflow: hidden.

List-styling

list-style-type — disc, circle, square, decimal, lower-roman, none osv.

list-style-position — inside/outside (om bullet er inden for eller uden for tekstflow).


CSS Counters

counter-reset: navn — nulstil tæller.

counter-increment: navn — tæl op ved hvert element.

content: counter(navn) — vis tæller i ::before.

/* Simpel list-styling */
ul.navliste { list-style: none; padding: 0; }

/* Custom bullet med ::before */
ul.check li::before {
  content: "✓ ";
  color: green;
  font-weight: bold;
}

/* CSS Counter — auto-nummerering */
ol.trin {
  list-style: none;
  counter-reset: trin-tæller;
}
ol.trin li {
  counter-increment: trin-tæller;
}
ol.trin li::before {
  content: counter(trin-tæller) ". ";
  font-weight: bold;
  color: royalblue;
}

/* Kapitelnummerering med indlejrede tællere */
body { counter-reset: kapitel; }
h2::before {
  counter-increment: kapitel;
  content: "Kapitel " counter(kapitel) ": ";
}
